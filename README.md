# Research Portfolio Website

A personal academic website with automatic publication syncing from ORCID and Google Scholar.

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| Structure | HTML5 | Page content and semantics |
| Style | CSS3 (no framework) | Layout, typography, animations |
| Behavior | Vanilla JavaScript | ORCID API integration, dynamic rendering |
| Hosting | GitHub Pages | Free static site hosting |
| Automation | GitHub Actions + Python | Weekly Google Scholar scraping |

---

## Quick Start

### 1. Create your GitHub repository

Your repository name determines your URL:
- Name it `yourusername.github.io` → site lives at `https://yourusername.github.io`
- Any other name → site lives at `https://yourusername.github.io/repo-name`

```bash
# Clone this template (or push these files to your new repo)
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/yourusername.github.io.git
git push -u origin main
```

### 2. Enable GitHub Pages

1. Go to your repo on GitHub
2. Settings → Pages
3. Source: **Deploy from a branch**
4. Branch: `main`, folder: `/ (root)`
5. Click Save — your site will be live in ~1 minute

### 3. Customize your content

Edit `index.html`:
- Replace `Your Full Name`, department, university
- Update `hero-bio` paragraph
- Change the ORCID, Google Scholar, GitHub, CV links
- Replace `photo.jpg` with your own photo
- Edit the three research area cards

Edit `script.js` — update the CONFIG object at the top:
```javascript
const CONFIG = {
  orcidId: "0000-0001-2345-6789",   // Your actual ORCID iD
  yourName: "Jane Smith",            // As it appears in author lists
  ...
};
```

### 4. Set up Google Scholar auto-updates

**a) Add your Scholar ID as a GitHub Actions Variable:**
1. GitHub repo → Settings → Secrets and variables → Actions → Variables tab
2. New variable: `GOOGLE_SCHOLAR_ID` = the ID from your Scholar URL
   - Your URL looks like: `scholar.google.com/citations?user=XXXXXXXX`
   - The `XXXXXXXX` part is your ID

**b) (Optional) Add ScraperAPI key for reliability:**
1. Sign up free at https://www.scraperapi.com (1000 req/month free)
2. Settings → Secrets → New secret: `SCRAPER_API_KEY`

**c) Trigger manually to test:**
GitHub repo → Actions → "Update Publications" → Run workflow

The action will run weekly (Mondays 6am UTC) and commit `data/publications.json` automatically.

## Customization Guide

### Changing colors

All colors are CSS custom properties in `style.css`:

```css
:root {
  --color-accent: #b5451b;   /* Change this to your preferred accent color */
  --color-bg:     #f8f7f4;   /* Page background */
  --color-text:   #1c1b1a;   /* Primary text */
}
```

### Adding a dark mode

Add this to `style.css`:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-bg:       #111110;
    --color-bg-alt:   #1a1a18;
    --color-surface:  #1e1e1c;
    --color-border:   #2e2e2c;
    --color-text:     #f0ede6;
    --color-text-muted: #8a8580;
  }
}
```

### Adding more sections

Copy an existing `<section>` block in `index.html` and add a matching nav link.
Common additions: News/Updates, Teaching, Software/Code, Awards.

## How the ORCID Integration Works

1. On page load, `script.js` calls `https://pub.orcid.org/v3.0/{your-id}/works`
2. ORCID returns a grouped list with one entry per paper
3. The script fetches full details (including authors) in batches of 50
4. Results are merged with any local `data/publications.json` (from Scholar)
5. Publications are rendered as cards with filter controls

ORCID data is always fresh (live API call). Scholar data updates weekly via GitHub Actions.

## Troubleshooting

**Publications not loading:**
- Open browser DevTools (F12) → Console tab — look for error messages
- Check that your ORCID iD in `script.js` is correct (format: `0000-0000-0000-0000`)
- Make sure your ORCID works are set to **public** visibility on orcid.org

**GitHub Pages not updating:**
- Actions tab → check the latest workflow run for errors
- GitHub Pages can take 1–2 minutes to redeploy after a commit

**Scholar scraping blocked:**
- Add a ScraperAPI key (free tier is sufficient)
- Alternatively, manually maintain `data/publications.json`
